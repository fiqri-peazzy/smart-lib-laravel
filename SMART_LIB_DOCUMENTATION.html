1. Pengenalan Sistem SIPERPUS Fasilkom

1.1 Apa itu SIPERPUS Fasilkom?

SIPERPUS Fasilkom adalah sistem manajemen perpustakaan digital yang dirancang khusus untuk Fakultas Ilmu Komputer. Sistem ini menggabungkan fitur-fitur manajemen buku fisik dengan koleksi digital dalam satu platform terintegrasi yang mudah digunakan.

Sistem ini memungkinkan mahasiswa, dosen, dan staff perpustakaan untuk mengelola peminjaman buku, booking, pembayaran denda, dan akses koleksi e-book dengan efisien. Semua proses dapat dilakukan melalui aplikasi web yang responsif dan user-friendly.

1.2 Tujuan & Manfaat Sistem

📌 Tujuan Utama:

  • Modernisasi pengelolaan perpustakaan dengan sistem digital
  • Meningkatkan efisiensi operasional staff perpustakaan
  • Memberikan kemudahan akses informasi bagi pengguna
  • Mengurangi waktu tunggu dan proses manual
  • Menyediakan statistik dan laporan real-time

Manfaat untuk Berbagai Pengguna:

Pengguna Manfaat
Mahasiswa Cari buku mudah, booking online, tracking peminjaman, akses e-book
Dosen Kuota peminjaman lebih tinggi, priority booking, extended loan period
Staff Perpustakaan Automasi proses, denda otomatis, laporan real-time, notifikasi user
Admin Kontrol penuh sistem, user management, analytics, konfigurasi

1.3 Fitur Utama Sistem

📚 Manajemen Koleksi

  • Katalog buku fisik dengan cover image dan metadata lengkap
  • Barcode & QR Code otomatis untuk setiap copy
  • Koleksi digital (E-Book, Jurnal, Skripsi, Modul, Paper)
  • Kategorisasi otomatis dan rekomendasi per program studi

🔄 Sistem Peminjaman

  • Request peminjaman online dengan validasi otomatis
  • Checkout via QR Code scanning
  • Tracking peminjaman real-time
  • Perpanjangan dengan validasi business rules
  • Auto-calculate durasi berbeda per role (Dosen vs Mahasiswa)

📅 Sistem Booking

  • Booking untuk buku yang sedang dipinjam
  • Notifikasi otomatis saat buku tersedia
  • Priority queue (Dosen lebih prioritas)
  • 3-hari pickup window

💰 Sistem Denda & Pembayaran

  • Auto-calculate denda overdue (Rp 5.000/hari)
  • Metode pembayaran: Cash, Transfer, QRIS
  • Pembayaran bisa di-waive oleh admin
  • Tracking pembayaran lengkap

👤 User Management

  • 4 Role dengan hak akses berbeda
  • Credit score tracking (0-100)
  • Status user (active, suspended, graduated)
  • Kuota peminjaman per role

2. Panduan Instalasi & Setup Sistem

2.1 Persyaratan Sistem

Hardware Minimum:

  • Processor: Intel Core i3 atau setara
  • RAM: 4 GB minimum (8 GB recommended)
  • Storage: 2 GB space minimum
  • Network: Internet connection untuk dependencies

Software Requirements:

  • PHP: versi 8.2 atau lebih tinggi
  • Composer: PHP Package Manager (terbaru)
  • Node.js: versi 16+ (untuk build assets)
  • MySQL/MariaDB: versi 5.7+
  • Text Editor: Visual Studio Code (recommended)
  • Server Local: Laragon atau XAMPP

2.2 Langkah-Langkah Instalasi

Step 1: Siapkan Local Server (XAMPP/Laragon)

⚠️ Cara menggunakan XAMPP:

  • Download XAMPP dari https://www.apachefriends.org
  • Install dengan default settings
  • Buka Control Panel XAMPP
  • Start layanan Apache dan MySQL
  • Pastikan port 3306 (MySQL) dan 80 (Apache) tidak terpakai

Atau menggunakan Laragon (Recommended):

  • Download Laragon dari https://laragon.org
  • Install dan buka aplikasi
  • Klik tombol "Start All" untuk menjalankan server
  • Laragon otomatis setup virtual host untuk project

Step 2: Clone atau Copy Project

Asumsikan project sudah berada di c:\xampp\htdocs\smart-lib (untuk XAMPP) atau folder htdocs di Laragon.

Step 3: Buka Terminal di VS Code

🔹 Cara membuka terminal di VS Code:

  1. Buka VS Code
  2. Klik menu File → Open Folder
  3. Pilih folder smart-lib
  4. Tekan Ctrl + ` (backtick) atau Ctrl + Shift + ` untuk buka terminal
  5. Jika ingin menggunakan Bash, klik dropdown terminal pilih bash

Step 4: Install PHP Dependencies (Composer)

Jalankan command berikut di terminal:

composer install

Proses ini akan mengunduh dan install semua library PHP yang diperlukan. Tunggu hingga selesai (biasanya 2-5 menit).

Step 5: Install Node.js Dependencies (NPM)

npm install

Atau jika menggunakan Yarn:

yarn install

Step 6: Setup Environment File

Copy file .env.example menjadi .env:

copy .env.example .env

Atau di terminal Linux/Mac:

cp .env.example .env

Step 7: Generate Laravel Application Key

php artisan key:generate

Proses ini akan menggenerate APP_KEY di file .env untuk encryption.

2.3 Konfigurasi Database

Langkah 1: Buat Database

Menggunakan phpMyAdmin (XAMPP):

  1. Buka browser dan akses http://localhost/phpmyadmin
  2. Login dengan user root (password kosong default)
  3. Klik menu Databases
  4. Isi nama database: smart_lib
  5. Pilih collation: utf8mb4_unicode_ci
  6. Klik Create

Langkah 2: Konfigurasi File .env

Edit file .env dan sesuaikan dengan konfigurasi database Anda:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=smart_lib
DB_USERNAME=root
DB_PASSWORD=

Penjelasan:

  • DB_HOST: Alamat server database (localhost = 127.0.0.1)
  • DB_PORT: Port MySQL (default: 3306)
  • DB_DATABASE: Nama database yang baru dibuat
  • DB_USERNAME: Username database (default: root)
  • DB_PASSWORD: Password database (kosong jika tidak ada)

Langkah 3: Jalankan Migration

Migration adalah perintah untuk membuat table-table database secara otomatis:

php artisan migrate

Proses ini akan membuat table berikut:

  • users - Data pengguna
  • books - Katalog buku
  • book_items - Copy/item buku fisik
  • book_categories - Kategori buku
  • loans - Data peminjaman
  • bookings - Data booking
  • fines - Data denda
  • digital_collections - Koleksi digital (e-book)
  • Dan table pendukung lainnya...

Langkah 4: Seed Database (Optional - untuk test data)

Jalankan seeder untuk mengisi data initial (opsional):

php artisan db:seed

Atau jika ingin seed seeder tertentu:

php artisan db:seed --class=BookSeeder

2.4 Build Frontend Assets

Kompilasi CSS dan JavaScript:

npm run build

Atau untuk development mode dengan auto-reload:

npm run dev

2.5 Menjalankan Server

Buka terminal baru (jangan tutup terminal yang sudah run) dan jalankan:

php artisan serve

Atau dengan custom host dan port:

php artisan serve --host=0.0.0.0 --port=8000

2.6 Akses Aplikasi

URL Akses Keterangan

http://localhost:8000/admin Admin Panel Filament CMS untuk management
http://localhost:8000/login Login Halaman login user

✅ Setup Selesai!

3.2 Model Data Utama (Entity Relationship)

Penjelasan Hubungan Antar Tabel:

🔹 Hubungan Buku (Book)

Tujuan: Menyimpan informasi katalog buku. Satu book bisa punya banyak copy (book_items).

Relasi:

  • Book → BookItem: One-to-Many (1 buku bisa punya N copy)
  • Book → BookCategory: Many-to-Many (1 buku bisa punya N kategori)

🔹 Hubungan Peminjaman (Loan)

Tujuan: Mencatat setiap transaksi peminjaman buku. Melacak siapa yang meminjam, kapan, sampai kapan, dan status.

Relasi:

  • Loan → User: Many-to-One (N peminjaman dari 1 user)
  • Loan → BookItem: Many-to-One (N peminjaman dari 1 copy buku)
  • Loan → Fine: One-to-One (1 peminjaman bisa punya 1 denda ketika overdue)

🔹 Hubungan Booking

Tujuan: Mencatat antrian ketika buku sedang dipinjam user lain.

Relasi:

  • Booking → User: Many-to-One (N booking dari 1 user)
  • Booking → Book: Many-to-One (N booking untuk 1 buku)

🔹 Hubungan Denda (Fine)

Tujuan: Mencatat denda ketika peminjaman overdue dan tracking pembayaran.

Relasi:

  • Fine → Loan: One-to-One (1 denda dari 1 peminjaman)
  • Fine → User: Many-to-One (N denda dari 1 user)

4. Pengguna & Peran Sistem (Role Management)

4.1 Empat Role Utama Sistem

SIPERPUS Fasilkom memiliki 4 role utama dengan hak akses dan fitur yang berbeda sesuai dengan kebutuhan mereka:

4.2 Role: ADMIN

👨‍💼 Admin - Pimpinan Perpustakaan

Deskripsi: Memiliki akses penuh ke semua fitur sistem. Bertanggung jawab atas konfigurasi dan manajemen keseluruhan.

Hak Akses:

  • ✓ Akses Admin Panel (Filament CMS)
  • ✓ Manajemen User (CRUD, assign role)
  • ✓ Manajemen Buku & Koleksi (CRUD lengkap)
  • ✓ Manajemen Peminjaman (override, modify)
  • ✓ Manajemen Denda (waive, modify)
  • ✓ View Laporan & Analytics
  • ✓ Konfigurasi Sistem
  • ✓ Audit Log & History

Workflow Utama:

  1. Login dengan credentials admin
  2. Akses Admin Panel di /admin
  3. Kelola resource melalui Filament interface
  4. View analytics dan laporan
  5. Configure sistem settings

4.3 Role: STAFF PERPUSTAKAAN

👨‍💻 Staff Perpustakaan - Operator

Deskripsi: Mengelola operasional harian perpustakaan. Handle proses peminjaman, pengembalian, booking, dan denda.

Hak Akses:

  • ✓ Access Admin Panel (read-only untuk beberapa bagian)
  • ✓ View & Process Loans (checkout, return)
  • ✓ View & Process Bookings
  • ✓ Manage Fines (record payment, waive dengan approval)
  • ✓ View Book Inventory
  • ✓ Generate Reports
  • ✗ Tidak bisa edit user
  • ✗ Tidak bisa konfigurasi sistem

Workflow Utama:

  1. User datang ke perpustakaan dengan QR code
  2. Staff scan QR code untuk verify request
  3. Staff process checkout (set loan_date)
  4. Saat user return: record return & kondisi buku
  5. Sistem auto-calculate denda jika overdue
  6. Record pembayaran denda
  7. Generate laporan harian/bulanan

4.4 Role: DOSEN

👨‍🏫 Dosen - Pengajar/Akademisi

Deskripsi: Pengguna dengan privilege lebih tinggi dibanding mahasiswa. Mendapat benefit special untuk penelitian dan pengajaran.

Hak Akses:

  • ✓ Access User Dashboard
  • ✓ Search & Browse Katalog Buku
  • ✓ Request/Pinjam Buku (PRIORITAS TINGGI)
  • ✓ Booking Buku (PRIORITY QUEUE)
  • ✓ Access Koleksi Digital
  • ✓ View Profile & Riwayat Peminjaman
  • ✓ Perpanjangan Buku (Extended Duration)
  • ✗ Tidak bisa akses admin panel

Privilege Khusus Dosen:

Privilege Dosen Mahasiswa
Durasi Peminjaman 21 hari (3 minggu) 14 hari (2 minggu)
Max Loan (Kuota) 10 buku 3 buku
Perpanjangan 2x (bisa extend 2 kali) 1x (hanya 1x extend)
Booking Priority HIGH (posisi awal) NORMAL (first-come)
Late Fee Waive Bisa dipertimbangkan admin Harus bayar penuh

Workflow Peminjaman Dosen:

  1. Login ke dashboard user
  2. Search buku di katalog
  3. Click "Request Buku" atau "Booking"
  4. Sistem akan prioritize request (durasi lebih lama)
  5. Dapatkan notif untuk pickup
  6. Bisa perpanjang 2x selama belum overdue
  7. Return buku sesuai due date

4.5 Role: MAHASISWA

👨‍🎓 Mahasiswa - User Utama

Deskripsi: Role default untuk pengguna regular. Bisa meminjam buku dengan kuota dan durasi standar.

Hak Akses:

  • ✓ Access User Dashboard
  • ✓ Search & Browse Katalog Buku
  • ✓ Request/Pinjam Buku (STANDARD)
  • ✓ Booking Buku (FIRST-COME BASIS)
  • ✓ Access Koleksi Digital
  • ✓ View Profile & Riwayat Peminjaman
  • ✓ Perpanjangan Buku (Limited Duration)
  • ✗ Tidak bisa akses admin panel

Validasi Peminjaman Mahasiswa:

  • ✓ Status: Active (tidak suspended/graduated)
  • ✓ Total Fines: = 0 (semua denda sudah lunas)
  • ✓ Active Loans: < 3 (belum capai kuota)
  • ✓ Credit Score: >= 50 (kredibilitas cukup)

Workflow Peminjaman Mahasiswa:

  1. Login dengan NIM & password
  2. Browse katalog buku atau search
  3. Click "Request" pada buku yang ingin dipinjam
  4. Sistem validasi (cek kuota, denda, status)
  5. Jika OK: Generate QR code, terima notif
  6. Pergi ke perpustakaan dengan QR code
  7. Staff scan QR, terima buku
  8. Punya 14 hari untuk return (atau perpanjang 1x)
  9. Return buku, settle denda jika ada

4.7 Permission Model (Spatie Laravel Permission)

System menggunakan Spatie Laravel Permission package untuk granular permission control. Setiap role memiliki permissions yang tersusun dari:

🔹 Permission Struktur:

  • view_* - Hak untuk melihat resource
  • create_* - Hak untuk membuat resource baru

  • delete_* - Hak untuk delete resource
  • *_any - Bisa action resource milik user lain
  • *_own - Hanya bisa action resource milik sendiri

5.2 Item/Copy Buku & QR Code Integration

Konsep Item Buku:

🔹 Perbedaan Book vs BookItem:

  • Book = Katalog/metadata buku (judul, penulis, penerbit, dll) - Hanya 1 record per judul unik
  • BookItem = Physical copy/eksemplar buku - Bisa banyak (misal, 3 copy dari buku yang sama)

✅ Contoh Praktis:

Book: "Clean Code" by Robert Martin (hanya 1 record)

BookItem:

  • Copy #1: Barcode BOOK-2024-00001, QR Code {QR-1}, Status: Available
  • Copy #2: Barcode BOOK-2024-00002, QR Code {QR-2}, Status: On Loan
  • Copy #3: Barcode BOOK-2024-00003, QR Code {QR-3}, Status: Damaged

Data Item Buku:

Field Tipe Deskripsi
Barcode String (Auto) Format: BOOK-YYYY-XXXXX (contoh: BOOK-2024-00001)
QR Code Image (Auto) Generate otomatis saat item dibuat

6.2 Status Peminjaman (State Machine)

Status Deskripsi Transisi Ke
pending Request sudah dibuat, menunggu user pickup → active (setelah checkout) / → expired (jika timeout)
active Buku sedang dipinjam user → extended (jika diperpanjang) / → overdue (jika lewat due_date) / → returned (saat return)
extended Peminjaman telah diperpanjang → overdue (jika lewat new due_date) / → returned
overdue Buku belum dikembalikan setelah due_date → returned (saat return, fine akan auto-calculate)
returned Buku telah dikembalikan ke perpustakaan ----- (Final state)

6.3 Validasi & Business Rules Peminjaman

⚠️ Kondisi User BISA Pinjam:

✓ Status = 'active' (tidak suspended, graduated, atau inactive)
✓ Total unpaid fines = 0 (semua denda harus sudah lunas)
✓ Active loans < max_loans (tidak over quota):
  - Dosen: max 10 books
  - Mahasiswa: max 3 books
✓ Credit score >= 50 (kredibilitas minimal)
✓ Minimal 1 copy buku tersedia (status = 'available')

⚠️ Kondisi User BISA Perpanjang:

✓ Loan status = 'active' (sedang dipinjam)
✓ is_extended = false (belum pernah extend)
✓ today < due_date (belum overdue)
✓ Tidak ada booking dari user lain untuk buku ini
✓ Max perpanjangan:
  - Dosen: 2x perpanjangan
  - Mahasiswa: 1x perpanjangan

✅ Durasi Peminjaman (Auto-Calculate):

Dosen:
  - Initial: 21 hari
  - Per extension: +7 hari (max 2x = 35 hari total)

Mahasiswa:
  - Initial: 14 hari
  - Per extension: +7 hari (max 1x = 21 hari total)

8. Sistem Denda & Pembayaran

8.1 Kalkulasi Denda Otomatis

Rumus Denda:

⚠️ Formula Perhitungan Denda:

DENDA = Days Overdue × Daily Rate

Contoh:
- Daily Rate: Rp 5.000 / hari
- Buku dikembalikan 3 hari terlambat
- Denda = 3 × Rp 5.000 = Rp 15.000

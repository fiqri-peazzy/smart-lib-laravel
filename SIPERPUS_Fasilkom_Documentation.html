1. Pengenalan Sistem SIPERPUS Fasilkom

1.1 Apa itu SIPERPUS Fasilkom?

SIPERPUS Fasilkom adalah sistem manajemen perpustakaan digital yang dirancang khusus untuk Fakultas Ilmu Komputer. Sistem ini menggabungkan fitur-fitur manajemen buku fisik dengan koleksi digital dalam satu platform terintegrasi yang mudah digunakan.

Sistem ini memungkinkan mahasiswa, dosen, dan staff perpustakaan untuk mengelola peminjaman buku, booking, pembayaran denda, dan akses koleksi e-book dengan efisien.

1.2 Tujuan & Manfaat

📌 Tujuan Utama:

  • Modernisasi pengelolaan perpustakaan dengan sistem digital
  • Meningkatkan efisiensi operasional staff perpustakaan
  • Memberikan kemudahan akses informasi bagi pengguna
  • Mengurangi waktu tunggu dan proses manual
  • Menyediakan statistik dan laporan real-time

1.3 Fitur Utama Sistem

Modul Deskripsi
📚 Manajemen Koleksi Katalog buku, barcode/QR code, koleksi digital
🔄 Peminjaman Request, checkout, return, perpanjangan
📅 Booking Antrian dengan priority system
💰 Denda Auto-calculate overdue + payment tracking
👤 User Management 4 role dengan hak akses berbeda

1.4 Teknologi yang Digunakan

Komponen Teknologi

Database MySQL / MariaDB
Admin Panel Filament CMS v3.3+
Frontend Blade Templates + Alpine.js

2. Panduan Instalasi & Setup

2.1 Persyaratan Sistem

Hardware:

  • Processor: Intel Core i3 atau setara
  • RAM: 4 GB minimum (8 GB recommended)
  • Storage: 2 GB space

Software:

  • PHP: 8.2 atau lebih tinggi
  • Composer: PHP Package Manager
  • Node.js: 16+
  • MySQL/MariaDB: 5.7+
  • Server Local: Laragon atau XAMPP

2.2 Langkah-Langkah Instalasi

Step 1: Siapkan Local Server

⚠️ Menggunakan XAMPP:

  • Download dari https://www.apachefriends.org
  • Install dengan default settings
  • Buka Control Panel XAMPP
  • Start layanan Apache dan MySQL

Atau menggunakan Laragon (Recommended):

  • Download dari https://laragon.org
  • Install dan buka aplikasi
  • Klik "Start All" untuk menjalankan server

Step 2-3: Buka Terminal di VS Code

Buka folder project dengan VS Code, tekan Ctrl + ` untuk buka terminal.

Step 4: Install Dependencies

composer install
npm install

Step 5: Setup Environment

copy .env.example .env
php artisan key:generate

Step 6: Konfigurasi Database

Edit file .env:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=smart_lib
DB_USERNAME=root
DB_PASSWORD=

Buat database di phpMyAdmin (localhost/phpmyadmin) dengan nama smart_lib.

Step 7: Run Migration

php artisan migrate
npm run build

Step 8: Jalankan Server

php artisan serve

Akses di http://localhost:8000 untuk frontend dan http://localhost:8000/admin untuk admin panel.

3.2 Model Data Utama

Hubungan Antar Entity:

🔹 Book (Katalog Buku)

Menyimpan data buku. Relasi: Book → BookItem (1 buku bisa punya N copy)

🔹 BookItem (Copy Fisik)

Setiap copy buku fisik dengan barcode & QR code unik

🔹 Loan (Peminjaman)

Mencatat transaksi peminjaman. Relasi: User → Loan ← BookItem

🔹 Booking (Antrian)

Booking untuk buku sedang dipinjam. Relasi: User → Booking ← Book

🔹 Fine (Denda)

Denda overdue. Relasi: Loan → Fine (1:1)

  • ✓ CRUD semua resource
  • ✓ Assign role & permission

👨‍💻 Operator

Mengelola operasional harian: checkout, return, booking, denda.

  • ✓ Process loans (checkout/return)
  • ✓ Manage bookings
  • ✓ Record fine payment
  • ✓ Generate reports

DOSEN

👨‍🏫 Pengajar

Privilege lebih tinggi: durasi pinjam 21 hari, kuota 10 buku, priority booking.

  • Durasi peminjaman: 21 hari (vs 14 hari mahasiswa)
  • Max loans: 10 buku (vs 3 buku mahasiswa)
  • Perpanjangan: 2x (vs 1x mahasiswa)

MAHASISWA

👨‍🎓 User Standar

Role default. Durasi pinjam 14 hari, kuota 3 buku, first-come-first-serve.

  • Durasi peminjaman: 14 hari
  • Max loans: 3 buku
  • Perpanjangan: 1x
  • Booking priority: NORMAL

6. Alur Peminjaman Lengkap (Loan Flow)

6.1 Workflow Lengkap: Request → Checkout → Return

┌─ REQUEST PEMINJAMAN ─┐ │ User browse buku → Click request │ Sistem validasi (status, denda, kuota) │ Create loan record → Status: pending │ Generate QR code → Send email notif └──────────┬──────────┘ │ ┌──────▼───────┐ │ CHECKOUT │ │ User ke perpus │ Staff scan QR │ Set loan_date → Status: active └──────┬───────┘ │ ┌──────▼──────┐ │ DURING LOAN │ │ 14-21 days │ Option: EXTEND │ Check: OVERDUE? └──────┬──────┘ │ ┌──────▼───────┐ │ RETURN │ │ User kembali │ Staff inspect │ Record condition │ Set return_date │ Status: returned └──────┬───────┘ │ ┌──────▼────────┐ │ AUTO-CHECK │ │ Fine generated │ Booking notified │ Process complete └───────────────┘

6.2 Status Peminjaman (State Machine)

Status Deskripsi
pending Menunggu user pickup
active Buku sedang dipinjam
extended Peminjaman diperpanjang
overdue Belum return setelah due_date
returned Sudah dikembalikan (final)

6.3 Validasi & Business Rules

⚠️ User BISA Pinjam Jika:

  • ✓ Status = 'active' (tidak suspended)
  • ✓ Total fines = 0 (semua denda lunas)
  • ✓ Active loans < max_loans
  • ✓ Credit score >= 50
  • ✓ Copy tersedia

⚠️ User BISA Perpanjang Jika:

  • ✓ Status = 'active'
  • ✓ Belum pernah extend
  • ✓ Belum overdue
  • ✓ Tidak ada booking dari user lain

✅ Durasi Peminjaman:

  • Dosen: 21 hari (extend +7 hari, max 2x)
  • Mahasiswa: 14 hari (extend +7 hari, max 1x)

8. Sistem Denda & Pembayaran

8.1 Kalkulasi Denda Otomatis

⚠️ Rumus Denda:

DENDA = Days Overdue × Rp 5.000/hari

Contoh: Terlambat 3 hari = 3 × Rp 5.000 = Rp 15.000

Cara Kerja Auto-Generate:

  • System job scheduler berjalan setiap hari
  • Check semua loan dengan status active & overdue
  • Calculate days_overdue
  • Create Fine record & send email notif
  • User tidak bisa pinjam sampai denda lunas

8.2 Metode Pembayaran

Metode Proses
Cash User bayar langsung ke perpustakaan
Transfer User transfer ke rekening perpustakaan
QRIS/E-Wallet Staff generate QRIS, user scan & bayar

Proses pembayaran: User bayar → Staff input di sistem → Fine status = 'paid' → User bisa pinjam lagi

8.3 Impact Denda terhadap User

  • ❌ TIDAK BISA PINJAM jika ada unpaid fine
  • ❌ TIDAK BISA BOOKING jika ada unpaid fine
  • ⚠️ CREDIT SCORE TURUN saat overdue
  • 📝 LAPORAN KE PIMPINAN jika berulang
  • ✅ NORMAL LAGI setelah denda paid/waived

10.4 Special: QR Code Checkout/Return

Checkout Process:

  1. Buka "Loans" filter pending
  2. Pilih loan → Click "Checkout"
  3. Sistem scan/verify QR code
  4. Confirm loan_date & due_date
  5. Click "Confirm" → Status = active

Return Process:

  1. Buka "Loans" filter active
  2. Pilih loan → Click "Return"
  3. Input condition (dropdown)
  4. Click "Confirm" → Status = returned
  5. Auto-check fine jika overdue

11. Troubleshooting & FAQ

11.1 Pertanyaan Umum (FAQ)

❓ Durasi peminjaman?

A: Mahasiswa: 14 hari, Dosen: 21 hari. Bisa diperpanjang dengan syarat.

❓ Kuota maksimal?

A: Mahasiswa: 3 buku, Dosen: 10 buku.

❓ Booking prioritas?

A: YA, Dosen di-notify duluan (priority queue).

❓ Berapa denda?

A: Rp 5.000 per hari. Auto-generated saat overdue.

❓ Bayar denda bagaimana?

A: Cash, transfer, atau QRIS. Staff input di sistem.

❓ Bisa pinjam kalau ada denda?

A: TIDAK. Harus lunas dulu.

❓ Akses e-book?

A: Langsung baca via browser PDF viewer atau download.

11.2 Troubleshooting Teknis

Problem: "Database not found"

⚠️ Solusi:

  • Pastikan MySQL running
  • Check .env: DB_DATABASE=smart_lib
  • Create database di phpMyAdmin
  • Run: php artisan migrate

Problem: Admin Panel error 500

⚠️ Solusi:

  • Check logs: storage/logs/laravel.log
  • Run: php artisan cache:clear
  • Run: npm run build
  • Restart server

Problem: QR Code tidak ter-generate

⚠️ Solusi:

  • Check: composer require endroid/qr-code
  • Check storage permission
  • Run: php artisan storage:link

Problem: Email notifikasi tidak terkirim

⚠️ Solusi:

  • Default: MAIL_MAILER=log (untuk development)
  • Check email di: storage/logs/laravel.log
  • Untuk production: Config SMTP (Gmail/Mailtrap)
